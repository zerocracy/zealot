# Zealot Plugin Repository

Each rule or paragraph in a `SKILL.md` file must be a single sentence;
  use semicolons, colons, or em dashes to chain related clauses inside
  one sentence, but never start a second sentence inside the same
  paragraph — split it into a new paragraph instead.

Keep every rule concise and specific: name the action, the target, and
  the condition, and stop there.

Formulate rules as direct orders, not opinions, suggestions, or
  preferences (`Run the full build`, not `It is usually a good idea
  to run the full build`).

Strip emotions and feelings from rule text: no `we love`, `we hate`,
  `unfortunately`, `sadly`, `please`, `kindly`, `we feel`, exclamation
  marks, or rhetorical flourish — the agent reads orders, not prose.
